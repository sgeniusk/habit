# Persona Habit i18n Plan

Updated: 2026-05-15

## 방향

Persona Habit은 한국어 감성 UX를 우선으로 만들되, 알파 단계부터 영어와 이후 일본어 확장을 염두에 둔다. 지금 목표는 모든 문장을 완역하는 것이 아니라, 핵심 제품 언어가 코드 안에 흩어지지 않게 만드는 것이다.

## 현재 구현

- 기본 언어는 `ko`다.
- 지원 언어 타입은 `Locale = "ko" | "en"`으로 시작한다.
- 번역 사전은 `src/lib/i18n.ts`에 둔다.
- 사용자의 언어 선택은 `UserPreferenceState.locale`에 저장된다.
- 잘못 저장된 언어 값은 한국어로 되돌린다.
- `document.documentElement.lang`을 현재 언어와 맞춘다.
- 우선 분리한 영역:
  - 하단 탭 이름
  - 언어 선택 UI
  - 오늘 화면 핵심 제목과 스냅 CTA
  - 스냅 화면 제목, 저장 버튼, 카테고리, 장소, 인증 도장, 메모 입력
  - 기록 행의 카테고리/장소 라벨

## 아직 남은 영역

- 날씨/위치 문구
- 일기 대화 문구
- 페르소나 이름, 직업, 진화명, 말투
- 모임 추천과 초대 플로우 문구
- 리포트/오래된 기억/AI 인사이트 문구
- 날짜, 시간, 거리, 온도 단위 포맷
- 영어 UX 감성 검수

## 원칙

- 한국어 문구를 먼저 제품적으로 다듬고, 그 의미를 영어 키에 맞춰 옮긴다.
- 화면 컴포넌트에 새 핵심 문구를 직접 추가하지 않는다.
- 번역 키는 기능 단위로 읽히게 작성한다.
- 페르소나 말투와 AI 인사이트는 직역하지 않고 언어별 톤을 따로 설계한다.
- 의료, 건강, 진단처럼 오해될 수 있는 문구는 언어별 안전 문구 기준을 따로 둔다.

## 다음 단계

1. 모임/리포트 화면의 CTA와 상태 문구를 i18n 키로 이동한다.
2. 페르소나 이름/직업/진화명을 locale-aware 데이터로 분리한다.
3. 날짜/시간/거리/온도 formatter를 `src/lib/i18n.ts` 또는 별도 `formatters.ts`로 분리한다.
4. 영어 화면에서 긴 단어가 버튼과 카드 밖으로 밀리지 않는지 시각 QA를 추가한다.
5. 이후 일본어를 넣을 때는 먼저 핵심 용어집을 만든다.
