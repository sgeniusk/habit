# Formi — 디자인 의뢰 브리프

Formi 앱의 비주얼·UX를 외부(Claude 디자인 세션 또는 디자이너)에 의뢰할 때 쓰는 문서다.
프롬프트를 그대로 복사해 쓰고, 라이브 URL과 첨부 자료를 함께 준다.

## 라이브 데모

- 공개 URL: https://sgeniusk.github.io/habit/
- GitHub Pages(`gh-pages` 브랜치)로 배포된 Expo 웹 빌드. 브라우저로 5탭을 다 볼 수 있다.
- 카메라·위치는 웹에서 제한되지만 화면·캐릭터·날씨·방·필터 디자인 검토에는 충분하다.

## 의뢰 방법

1. 새 Claude 세션(디자인 작업용)을 연다.
2. 아래 **프롬프트 1**(전체 앱 리디자인)을 붙이고, 라이브 URL을 같이 준다.
3. 방 꾸미기는 별도 비중이 크므로 **프롬프트 2**(아이소메트릭 방)를 이어서 준다.
4. 가능하면 첨부한다 — 현재 5탭 스크린샷, Formi 캐릭터 PNG(`mobile/assets/formi/`),
   방 꾸미기 레퍼런스(아이소메트릭 방 데코 게임 스크린샷).
5. "진단 → 방향 → 구체안" 순서로 받고, 화면별로 한 탭씩 깊게 반복한다.

## 산출물을 받을 때 확인할 것

- React Native(Expo)로 구현 가능한가 — reanimated·react-native-svg 범위 안.
- 디자인 토큰(색·타이포·여백·radius·그림자)이 명세로 나왔는가.
- 아이소메트릭 가구 스프라이트가 동일 투영각·스케일·조명으로 일관된가.
- 정적 자산 네이밍·익스포트 규칙이 RN `require` 등록에 맞는가.

## 프롬프트 1 — 전체 앱 감성 리디자인

```
너는 iOS·Android 네이티브 앱을 전문으로 하는 시니어 프로덕트 디자이너야.
아래 앱의 현재 인터페이스를 직접 열어 보고, 모바일 앱의 감성을 제대로 살린
비주얼·UX 리디자인을 해줘.

## 앱 — Formi

"form + me". 정체성 없는 말랑한 캐릭터(Formi)가 사용자의 생활 기록을 먹고
형태를 얻어 자라는 습관 앱이다.
- 사용자가 일상의 한 컷을 사진으로 남기면(스냅) → 그 분야의 페르소나 캐릭터가
  단계적으로 성장한다 (씨앗 → 새싹 → 정체성 → 마스터).
- 분야: 공부 / 운동 / 식단 / 독서 / 정리 / 셀프케어 / 취미.
- 한국어 우선. 핵심 정서는 "따뜻함과 다정함" — 다그치는 습관앱이 아니라,
  곁에서 같이 자라는 동반자. 기록은 성취가 아니라 애착의 대상이다.

## 현재 인터페이스 — 직접 열어서 봐줘

https://sgeniusk.github.io/habit/

5개 탭으로 구성:
- 오늘 — 연속 기록(streak), 대표 페르소나, 날씨 풍경 애니메이션,
  페르소나의 챙김 한마디, 오늘 남긴 기록 목록
- 스냅 — 사진 선택 → 카테고리·장소·메모·필터·스티커·인증 도장 → 저장
- 집 — 보유 페르소나 컬렉션, 페르소나의 방(꾸미기), 말투 톤
- 모임 — AI 모임 제안, 초대 링크
- 리포트 — 주간 AI 인사이트, 오래된 기억 회상

현재는 기능 위주로 만들어져 비주얼이 투박하다. 카드가 평면적이고
여백·타이포·색·모션이 정서를 못 살린다. 시작점 색은 크림 배경(#f4f7e8)에
잎사귀 그린 포인트(#2f9d65) 정도.

## 플랫폼 / 기술 제약

- React Native (Expo) 네이티브 앱. 위 URL은 웹 데모지만 실제 타깃은
  iOS·Android 모바일 앱이다. 모바일 앱다운 제스처·탭바·세이프에어리어를
  전제로 설계해줘.
- 디자인은 React Native로 구현 가능해야 한다 — reanimated(애니메이션),
  react-native-svg(벡터·그라데이션) 사용 가능. 웹 전용 과한 CSS 효과는 안 됨.
- 유지할 것: Formi 캐릭터 컨셉, 5탭 구조, 한국어. 이 위에서 비주얼 언어와
  화면 흐름을 새로 설계한다.

## 원하는 방향 — 감성적인 모바일 앱

- 따뜻하고 손에 잡히는 질감. 종이·천·햇살 같은 아날로그 감성.
- 차분하고 여백이 넉넉한 레이아웃. 정보를 쏟지 않고 한 호흡씩 보여주기.
- Formi 캐릭터가 화면의 정서적 중심이 되도록.
- 게임처럼 다그치는 게이미피케이션이 아니라, 다이어리·편지 같은 다정한 톤.
- 부드러운 모션 — 등장·전환·캐릭터 idle이 살아있되 과하지 않게.

## 산출물

1. 디자인 방향성 — 무드 키워드, 컨셉, 레퍼런스 톤
2. 디자인 토큰 — 컬러 팔레트, 타이포 스케일, 여백 시스템, radius,
   그림자/elevation 단계
3. 화면별 리디자인 — 5탭 각각의 레이아웃·구성·위계 제안 (목업 또는 상세 설명)
4. 핵심 컴포넌트 스펙 — 카드, 버튼, 칩, 하단 탭바, 캐릭터 영역,
   날씨 영역, 기록 카드
5. 모션·인터랙션 가이드 — 화면 전환, 캐릭터 idle, 마이크로 인터랙션
6. Formi 캐릭터를 각 화면에서 감성적으로 배치·연출하는 방법

각 제안은 React Native(Expo)로 구현 가능한 형태로, 짧은 구현 메모를 곁들여줘.
먼저 위 URL을 열어 현재 상태를 진단한 뒤, 진단 → 방향 → 구체안 순서로 정리해줘.
```

## 프롬프트 2 — 핵심 기능: 아이소메트릭 방 꾸미기

```
## 핵심 기능 — 페르소나의 방 (아이소메트릭 방 꾸미기)

"집" 탭의 방은 단순 배경 선택이 아니라, 사용자가 가구를 직접 배치해
페르소나의 공간을 만드는 아이소메트릭(2.5D) 방 꾸미기 시스템이다.
참고 톤: 아늑한 방 데코 게임 (3/4 부감 아이소메트릭, 다이아몬드 바닥,
가구를 한 점씩 놓아 방을 채우는 류).

### 동작 방식
- 방은 아이소메트릭 시점의 한 칸짜리 공간. 바닥(다이아몬드 그리드) + 두 벽면.
- 사용자가 아이템 트레이에서 가구를 골라 방에 배치·이동한다.
- 페르소나(Formi 캐릭터)가 그 방 안에서 산다 — idle 애니메이션.
- 페르소나가 성장하거나 스냅을 쌓으면 새 아이템이 해금/구매된다 (상점·보상).
- 분야별 페르소나마다 자기 방을 가진다 (공부방·운동방·셀프케어방 등 결).

### 디자인해야 할 것
1. 방 셸 — 아이소메트릭 바닥/벽의 비주얼, 그리드 규격, 시점 각도 확정
2. 가구 아이템 세트 — 침대·책상·의자·소파·러그·화분·책장·조명·벽액자·
   소품 등. 분야 결을 살린 변형 포함. 전부 동일한 아이소메트릭 투영각·
   스케일 그리드·조명 방향·라인 두께로 그려진 일관 스프라이트 세트.
3. 아이템 트레이 / 배치 UI — 카테고리 탭, 드래그 또는 탭 배치, 이동·회전·제거
4. 상점 — 아이템 잠금/해금, 보유 재화, 페르소나 성장과 연결된 해금 규칙
5. 빈 방 → 꾸며진 방까지의 정서적 동선 (처음 받았을 때, 채워갈 때)
6. 아트 산출 규격 — 스프라이트 해상도, 그리드 셀 픽셀 크기, 투영 각도(예:
   2:1 아이소), 네이밍·익스포트 규칙 (React Native 정적 require 로 등록 가능하게)

### 제약
- React Native(Expo)로 구현. 가구는 투명 PNG 스프라이트, 코드가 그리드
  좌표에 절대배치한다. 스프라이트 세트는 각도·스케일이 픽셀 단위로 일관돼야
  코드가 정렬할 수 있다.
- Formi 캐릭터는 기존 것을 쓴다 (이미 있음). 방·가구만 새로 디자인.
- 코드 쪽 아이소메트릭 배치 엔진은 이미 구현 중이다 (바닥 그리드, 가구
  배치·깊이 정렬, 아이템 트레이). 디자이너는 그 엔진에 끼울 스프라이트 세트와
  비주얼 규격을 만들면 된다.
```

## 코드 쪽 진행 상황

- 아이소메트릭 방 배치 엔진을 React Native로 구현 중 (`mobile/src/components/IsoRoom.tsx`).
  바닥 다이아몬드 그리드, 타일 좌표 변환, 가구 배치·깊이 정렬, 아이템 트레이,
  배치/제거, 레이아웃 영속화. 가구는 임시 도형이며 디자이너 스프라이트로 교체한다.
- 스프라이트 교체 규격(타일 픽셀 크기·투영각)은 엔진 코드의 상수와 맞춘다.
