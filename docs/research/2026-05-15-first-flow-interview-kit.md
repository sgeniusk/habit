# 첫 흐름 사용자 인터뷰 준비 키트

Updated: 2026-05-15
Target: 3 명의 실제 사용자
Duration: 1 인당 40~45 분 (디바이스 시연 30 분 + 회고 15 분)
URL: <https://habit-six-gamma.vercel.app/>

## 검증할 가설

[docs/superpowers/specs/2026-05-13-persona-habit-prd-ko.md](../superpowers/specs/2026-05-13-persona-habit-prd-ko.md) 의 성공 기준 5 개를 검증 대상으로 본다.

1. 사용자가 10 초 안에 "생활 스냅 → 페르소나 성장" 관계를 이해한다.
2. 사용자가 공부/식단/운동 같은 생활 스냅을 자연스럽게 상상한다.
3. 여러 페르소나 보유, 성장, 꾸미기, 진화가 눈에 보인다.
4. 모임이 단순 채팅이 아니라 스냅 지속을 돕는 소셜 루프로 느껴진다.
5. AI 인사이트가 생활 기록과 건강한 습관 개선을 명확히 연결한다.

[docs/alpha/persona-habit-10-tester-evaluation-report.md](../alpha/persona-habit-10-tester-evaluation-report.md) 의 가상 평가 결과가 실제 사용자에게 그대로 나타나는지 확인한다. 가상 평가가 정량 점수를 부풀렸을 가능성이 있으므로 정성 응답 비중을 높게 본다.

## 모집 기준

10 명 가상 테스터 평가 보고서의 페르소나 5 군 중 다음을 선호한다.

- 학생 또는 수험생 1 명 (공부 동기)
- 직장인 또는 자영업자 1 명 (식단/운동 균형)
- 친구 그룹과 가볍게 기록하는 사람 1 명 (모임 루프 검증)

연령은 18~35 세, 모바일 웹 사용에 익숙한 사람으로 한정한다. 디자이너/개발자처럼 앱 도메인을 깊이 아는 사람은 한 명까지만 포함한다.

## 사전 동의와 데이터 처리

- 음성 녹음과 화면 녹화 동의를 인터뷰 전에 받는다.
- 녹화 파일은 인터뷰 종료 후 30 일 안에 삭제한다.
- 응답은 가명 처리해서 보고서에 인용한다.
- 인터뷰 인센티브 (커피 기프티콘 등) 는 인터뷰 직후 지급한다.

## 시연 환경

- 디바이스: 사용자의 평소 휴대폰 (가능한 한 본인 기기) + iOS Safari / Android Chrome.
- 화면 녹화: 디바이스 자체 녹화 또는 Zoom 공유.
- 보조 장비: 사용자가 보는 화면을 인터뷰어가 같이 보도록 미러링.
- 준비물: 인터뷰어 노트북, 노트, 피드백 시트 (아래 양식) 복사본 3 부.

## 인터뷰 흐름 (40~45 분)

### Phase 0. 인사와 동의 (5 분)

스크립트.

> "오늘 함께해주셔서 고마워요. 저는 Persona Habit 이라는 생활 스냅 앱을 만들고 있고, 오늘은 첫 흐름이 사용자에게 어떻게 보이는지 직접 확인하려고 합니다. 정답을 찾는 게 아니라 솔직한 첫 반응이 가장 도움이 돼요. 화면 녹화와 음성 녹음을 켜고 시작해도 괜찮을까요?"

확인 사항.

- 평소 생활기록/습관/일기 앱을 쓰는지.
- 마지막으로 쓴 비슷한 앱이 무엇인지, 왜 멈췄는지.

### Phase 1. 첫 30 초 관찰 (5 분)

설정.

- 사용자 디바이스에서 새 탭으로 URL 진입.
- 인터뷰어는 말 없이 관찰만 한다.

관찰 포인트.

- 첫 가이드 팝업의 3 단계를 그대로 따라가는가, 건너뛰는가.
- "첫 스냅 찍기" 버튼을 발견하는 시간 (목표 10 초 내).
- 손가락이 멈추거나 두리번거리는 순간을 표시.

기록.

| 시각 | 행동 | 추정 감정 |
| ---- | ---- | --------- |
| 0:00 | 진입 |           |
| 0:10 |      |           |
| 0:30 |      |           |
| 1:00 |      |           |

### Phase 2. 첫 스냅 (10 분)

요청.

> "평소 같으면 오늘 어떤 순간을 사진으로 남길지 한 번 떠올려 보세요. 그 사진을 스냅으로 올려볼까요?"

관찰 포인트.

- 사진 선택 → 필터 → 스티커 → 도장 → 카테고리 → 장소 → 저장 흐름 중 어디서 멈추는가.
- "꾸며서 올리기" 버튼을 자연스럽게 누르는가.
- 저장 후 "집 / 모임 / 리포트" 빠른 이동 카드 중 어디로 가는가.
- 영어 모드 토글을 시도하는 사용자가 있는지 (자연스러운 발견 여부).

질문 (스냅 끝난 직후).

- "방금 한 흐름을 한 줄로 요약해 보세요."
- "저장하면 무엇이 바뀐다고 느끼나요?"
- "어디서 잠깐 멈췄거나 헷갈렸던 순간이 있나요?"

### Phase 3. 집 탭 (5 분)

요청.

> "방금 스냅이 어디로 갔는지 직접 확인해 볼까요?"

관찰 포인트.

- 대표 페르소나가 누구인지 사용자가 짚어내는가.
- "방 꾸미기" / "페르소나 꾸미기" / "말투/테마" 중 어떤 것을 먼저 만지는가.
- 페르소나 애칭을 바꾸는 사용자가 있는가.

질문.

- "이 페르소나는 누구라고 느껴지나요?"
- "방을 꾸민 결과가 다음 스냅을 더 남기고 싶게 만드나요?"
- "말투를 귀여운/차분한 사이에서 바꾸면 어떤 차이가 느껴지나요?"

### Phase 4. 모임 탭 (5 분)

요청.

> "혼자가 아니라 친구와 같이 한다면 어떤 모습일지도 한 번 볼까요?"

관찰 포인트.

- AI 모임 제안 카드를 어떻게 읽는가.
- "초대 링크 복사", "공유 문구 만들기" 까지 가는 사용자가 있는가.
- "초대 수락 미리보기" 의 의미를 즉각 이해하는가.

질문.

- "이 모임은 부담스러운가요, 가볍게 들어갈 수 있을까요?"
- "친구를 떠올렸을 때 누가 같이할 만한가요?"
- "이 모임이 있어서 다시 들어올 이유가 생기나요?"

### Phase 5. 리포트 탭 (5 분)

요청.

> "마지막으로 AI 가 무엇을 발견했는지 한 번 같이 봐요."

관찰 포인트.

- "AI가 발견한 숨은 습관" 카드를 읽는 시간.
- "근거" 라인을 발견하는가 (가상 평가에서 신뢰도 6.7 점이었음).
- "문구 순하게" / "관심 없음" / "오래된 기억" 모드 전환 중 어디까지 시도하는가.

질문.

- "이 분석이 맞다고 느끼나요?"
- "근거 한 줄이 분석의 신뢰도를 얼마나 바꾸나요?"
- "오래된 기억 모드는 어떤 상황에서 다시 보고 싶나요?"

### Phase 6. 회고와 자유 응답 (10 분)

질문.

1. "이 앱을 한 줄로 친구에게 설명한다면 뭐라고 말할까요?"
2. "내일 다시 들어올 이유 한 가지를 꼽는다면?"
3. "지금 가장 어색하거나 불편했던 순간은 무엇이었나요?"
4. "출시되면 1 천 원 정도의 가격을 낼 만한가요?"
5. "친구 한 명을 초대한다면 누구일까요?"
6. "이 앱이 1 개월 뒤에도 휴대폰에 남아 있을 것 같나요? 왜요?"

## 인터뷰어 행동 규칙

- 사용자가 헤맬 때 즉시 도와주지 않는다. 30 초 정도 더 지켜본다.
- "이게 무엇인 것 같나요?" 같은 열린 질문을 우선한다.
- 사용자가 한국어를 쓰면 한국어로, 영어를 쓰면 영어로 대응한다.
- 인터뷰어가 앱을 칭찬하지 않는다. 중립을 유지한다.
- 비판이 나오면 "왜 그렇게 느꼈는지" 까지 한 단계 더 묻는다.
- 인터뷰 끝에 인센티브와 결과 공유 일정을 명확히 안내한다.
