# Persona Animation Pipeline

Updated: 2026-05-14

## 결론

Persona Habit의 페르소나는 프로덕션에서 즉석 CSS 캐릭터나 AI가 만든 동물 일러스트를 그대로 쓰지 않는다. 앱은 캐릭터의 상태와 꾸미기 값을 관리하고, 실제 움직임은 전문 애니메이션 에셋으로 교체할 수 있게 만든다.

권장 순서:

1. **Rive**: 캐릭터 리깅, 타임라인, 상태머신을 한 파일에서 관리한다.
2. **Lottie**: After Effects 기반의 짧은 루프 모션이나 보상 이펙트에 쓴다.
3. **Spine**: 게임에 가까운 본 애니메이션과 많은 스킨 조합이 필요할 때 검토한다.

## 움직임은 어떻게 편집하는가

애니메이션 편집은 보통 세 단계로 나뉜다.

1. **파츠 분리**: 머리, 몸, 팔, 눈, 가방, 옷, 방 소품처럼 움직일 부분을 레이어나 본으로 나눈다.
2. **타임라인 편집**: idle, study, run, meal, sleep, celebrate 같은 루프를 키프레임으로 만든다.
3. **상태머신 연결**: 앱이 `activity=study`, `mood=celebrate`, `outfit=windbreaker` 같은 값을 넘기면 Rive/Lottie 런타임이 맞는 모션과 스킨을 보여준다.

즉, 개발자가 프레임을 직접 하나씩 움직이는 방식이 아니라 “앱 상태 → 애니메이션 상태머신 → 렌더링” 구조로 제어한다.

## 앱에서 관리할 상태

현재 프로토타입은 아래 상태를 먼저 관리한다.

- `activePersona`: 최신 스냅 카테고리에 따른 대표 페르소나
- `selectedRoomItem`: 방 꾸미기 선택값
- `selectedOutfitItem`: 페르소나 꾸미기 선택값
- `activity`: 공부, 운동, 식단, 독서, 정리, 셀프케어
- `rewardState`: 일반, 레벨업, 아이템 해금

나중에 Rive 파일이 들어오면 이 값들이 그대로 애니메이션 입력값이 된다.

## 에셋 제작 기준

- 동물형을 무리하게 직접 생성하지 않는다.
- 먼저 인간형, 마스코트형, 오브젝트형 중 하나의 스타일 가이드를 정한다.
- 페르소나별로 완전히 다른 캐릭터를 만들기보다, 하나의 기본 리그에 표정, 의상, 소품, 방 배경을 갈아 끼우는 구조를 우선한다.
- 앱 안에서는 캐릭터 이름, 활동, 방/의상 아이템 선택이 먼저 동작해야 하며, 비주얼 에셋은 교체 가능한 레이어로 취급한다.

## 핸드오프 포맷

디자인/애니메이션 작업자가 넘겨야 할 최소 항목:

- Rive 또는 Lottie 원본 파일
- 상태 이름 목록: `idle`, `study`, `run`, `meal`, `read`, `clean`, `selfcare`, `celebrate`
- 입력값 목록: activity, outfit, roomItem, rewardState
- 루프 여부와 전환 규칙
- 앱에서 표시할 fallback 정적 이미지

## 지금 프로토타입에서의 원칙

현재 CSS 페르소나는 동작 검증용 임시 표현이다. 이후 품질 단계에서는 캐릭터 아트와 움직임을 앱 코드에서 직접 그리지 않고, 에셋 툴에서 편집한 파일을 런타임으로 연결한다.
