# Persona Habit Roadmap

Updated: 2026-05-14

## 현재 상태 점검

Persona Habit은 현재 **모바일 웹 프로토타입** 단계다. 핵심 컨셉은 “생활 스냅을 남기면 페르소나가 성장하고, AI가 내가 모르는 습관까지 건강하게 읽어준다”로 정리되어 있다.

완료된 기반:

- `오늘 / 스냅 / 집 / 모임 / 리포트` 5탭 프로토타입
- 생활 스냅, 위치/날씨 카드, 일기 작성 방식 선택
- 사진 미리보기, 필터/스티커 오버레이, 새 기록 저장으로 이어지는 스냅 플로우
- 다양한 페르소나, 집 화면, 방/페르소나 꾸미기 방향
- 최신 스냅 카테고리에 따라 집 대표 페르소나가 바뀌는 반응 루프
- 규칙 기반 페르소나 성장 및 숨은 습관 인사이트
- `SnapRecord`, 샘플 기록, 페르소나 카탈로그 데이터 경계 분리
- `src/views/`, `src/components/` 기반 화면/공용 UI 컴포넌트 분리
- 한글/영문 PRD
- 제작 에이전트 조직도
- GitHub 공개 저장소, Vercel 배포, GitHub Actions CI
- 로컬 하네스: lint, format, test, build, audit

아직 비어 있는 핵심:

- 실제 사진 편집 결과물의 영구 저장과 공유용 이미지 내보내기
- 스냅 저장 데이터의 영속화
- 페르소나 카탈로그와 성장 엔진의 XP/해금 아이템 통합
- 위치/날씨 권한 플로우와 실패 상태
- 일기 작성 화면과 AI 보조 작성 흐름
- 모임 초대, 공동 페르소나, 그룹 스냅 루프
- 네이티브 iOS/Android 전환 계획

## 제품 원칙

1. **기록보다 스냅**: 사용자가 검사받는 느낌이 아니라 오늘의 순간을 가볍게 남기는 느낌이어야 한다.
2. **성장보다 정체성**: 레벨 숫자보다 “나는 어떤 생활 페르소나가 되어가고 있는가”가 먼저 보여야 한다.
3. **AI보다 건강한 해석**: AI는 단정하거나 진단하지 않고, 반복되는 환경과 선택을 부드럽게 비춰줘야 한다.
4. **경쟁보다 같이 하기**: 모임은 압박형 챌린지가 아니라 다시 돌아오기 쉬운 사회적 루프여야 한다.
5. **웹 프로토타입에서 네이티브로**: 지금은 흐름 검증, 이후 Expo 기반 iOS/Android로 확장한다.

## 단계별 로드맵

### Phase 0. Prototype Stabilization

목표: 현재 프로토타입을 다음 기능을 얹을 수 있는 구조로 정리한다.

상태: **데이터/화면 경계 안정화 완료**. `App.tsx`는 탭 상태와 기록 저장 흐름을 관리하고, 화면 구현은 `src/views/`, 반복 UI는 `src/components/`로 분리했다.

- `src/App.tsx`의 타입, 카탈로그, 샘플 상태를 분리
- `SnapRecord`를 공유 타입으로 정리하고 `VerificationRecord`는 임시 호환 alias로 유지
- 페르소나 카탈로그를 별도 데이터 모듈로 분리
- `오늘 / 스냅 / 집 / 모임 / 리포트` 화면을 뷰 컴포넌트로 분리
- 페르소나 아바타, 메트릭, 기록 행, 모임 행을 공용 컴포넌트로 분리
- 테스트가 탭, 스냅, 집, 리포트 핵심 흐름을 계속 보증하게 유지
- 문서와 하네스 상태를 최신화

완료 기준:

- 기존 UI는 유지된다.
- 테스트와 빌드가 통과한다.
- 다음 기능이 `App.tsx` 한 파일에 계속 누적되지 않는다.

### Phase 1. Snap Loop MVP

목표: 사용자가 스냅을 찍고 꾸미고 올렸다는 느낌을 만든다.

상태: **MVP 구현**. 사진 미리보기, 필터/스티커 선택, 프리뷰 오버레이, 새 기록 저장, 최근 기록 썸네일까지 동작한다.

- 이미지 선택 후 파일명뿐 아니라 미리보기 표시 완료
- 선택한 필터/스티커를 스냅 프리뷰 위에 시각적으로 표시 완료
- 필터 선택 상태와 스티커 선택 상태 구현 완료
- 선택한 이미지/필터/스티커/카테고리/장소/감정이 새 스냅 기록으로 저장 완료
- 최근 기록에 새 스냅이 즉시 반영 완료
- 사진 권한과 파일 선택 실패 상태 문구 정리

완료 기준:

- 사용자가 스냅을 남기면 오늘 기록, 리포트, 페르소나 성장에 반영된다.
- “꾸며서 올리기”가 단순 버튼이 아니라 실제 기록 루프로 느껴진다.

### Phase 2. Persona Home Loop

목표: 집 탭을 앱의 감정적 보상 중심으로 만든다.

상태: **부분 구현**. 최신 스냅 카테고리 기반 대표 페르소나 반응, 기록 기반 XP/레벨/해금 보상, 방/의상 꾸미기 선택 상태는 연결되었다. 하이브리드 페르소나와 에셋 교체형 애니메이션 연결이 남아 있다.

- 최신 스냅 카테고리에 따라 대표 페르소나 행동 변경 완료
- XP, 레벨, 해금 아이템을 실제 기록 기반으로 표시 완료
- 방 꾸미기와 페르소나 꾸미기 아이템 선택 상태 구현 완료
- Rive/Lottie/Spine 기반 애니메이션 에셋 파이프라인 문서화 완료
- 하이브리드 페르소나 조건 정의
- 페르소나 컬렉션을 카테고리별로 확장

완료 기준:

- 공부, 운동, 식단, 독서, 정리, 셀프케어 스냅이 각기 다른 페르소나 반응을 만든다.
- 집 탭에 들어갈 이유가 “구경”이 아니라 “내가 키운 결과 확인”이 된다.

### Phase 3. Today Journal And Context

목표: 오늘 탭을 하루 기록의 시작점으로 만든다.

- 위치/날씨 카드의 권한 요청, 허용, 거부, 실패 상태 설계
- 날씨 데이터 어댑터 인터페이스 초안
- AI랑 같이쓰기 / 혼자 기록하기 선택 이후 화면 추가
- 스냅, 날씨, 장소, 감정 기반 일기 초안 생성 흐름 설계
- Day One류 일기 앱을 참고한 타임라인/캘린더 진입 구조 설계

완료 기준:

- 오늘 탭이 단순 대시보드가 아니라 “오늘을 남기는 입구”가 된다.

### Phase 4. Meet Social Loop

목표: 스투디처럼 친구를 초대하고 같이 지속하는 유인을 만든다.

- 모임 생성/초대 링크 플로우 프로토타입
- 모임별 테마: 공부, 식단, 러닝, 혼합 생활
- 각자의 스냅이 공동 페르소나 XP에 기여
- 경쟁보다 복귀를 유도하는 가벼운 참여 상태
- 모임 리포트와 공동 페르소나 분위기 변화

완료 기준:

- 친구를 초대해야 할 이유가 보인다.
- 모임이 채팅방이 아니라 생활 스냅을 지속하게 하는 장치로 느껴진다.

### Phase 5. AI Habit Intelligence

목표: “내가 모르는 습관까지 AI가 발견한다”는 핵심 약속을 구현 가능한 형태로 좁힌다.

- 인사이트 타입: 장소 패턴, 시간대 패턴, 카테고리 균형, 회복 루틴, 하이브리드 정체성
- 안전 문구와 금지 문구 정의
- 사용자가 불편해할 수 있는 분석의 숨김/수정/피드백 플로우
- AI 일기 초안과 습관 인사이트의 데이터 경계 분리
- 실제 모델 연동 전 규칙 기반 시뮬레이션 강화

완료 기준:

- AI가 무섭거나 단정적으로 느껴지지 않고, 사용자가 스스로를 이해하는 데 도움을 준다.

### Phase 6. Native App Readiness

목표: 웹 프로토타입의 검증된 흐름을 iOS/Android 출시 계획으로 옮긴다.

- Expo React Native 구조 초안
- 카메라, 사진첩, 위치, 푸시 알림 권한 정책
- 로컬 저장, 계정, 백엔드 선택지 정리
- 앱스토어 제출 전 최소 기능 정의
- 웹과 네이티브의 공유 가능한 도메인 로직 분리

완료 기준:

- “웹에서 예쁜 프로토타입”을 넘어 실제 앱 출시 계획이 선다.

## 우선순위 판단

다음 개발 순서는 다음과 같다.

1. **구조 안정화**: 다음 기능이 들어갈 자리 만들기
2. **스냅 루프**: 사진을 남기고 꾸미고 기록되는 감각 만들기
3. **집 보상 루프**: 기록이 페르소나 행동과 꾸미기로 이어지게 만들기
4. **오늘 일기 루프**: 하루 회고와 AI 보조 작성으로 확장
5. **모임 루프**: 친구 초대와 공동 페르소나로 소셜 동기 강화
6. **AI/네이티브 준비**: 실제 서비스로 넘어가기 위한 기반 설계

| 위험                              | 현재 징후                           | 대응                                                |
| --------------------------------- | ----------------------------------- | --------------------------------------------------- |
| 기능이 한 파일에 계속 쌓임        | `App.tsx`의 화면 구현 분리는 완료됨 | 신규 기능은 `views/components/data/lib` 경계를 유지 |
| 스냅이 아직 진짜 사진 경험이 아님 | 파일명만 표시                       | Phase 1에서 이미지 미리보기와 꾸미기 상태 구현      |
| 페르소나 성장과 집 화면이 분리됨  | XP/레벨/해금/꾸미기 연결은 완료됨   | 애니메이션 에셋 상태머신과 연결                     |
| 위치/날씨가 하드코딩됨            | 서울 성수동, 18도 고정              | Phase 3에서 권한/어댑터/실패 상태 설계              |
| AI가 과장될 수 있음               | 현재 규칙 기반 문구만 있음          | Phase 5에서 안전 기준과 피드백 루프 먼저 정의       |
| 네이티브 전환 비용                | 웹 전용 구조가 커질 수 있음         | 도메인 로직을 UI 밖으로 분리                        |

## 다음 5개 실행 항목

1. 오늘 탭의 위치/날씨/일기 흐름을 권한 허용, 거부, 실패 상태별로 설계한다.
2. 스냅 편집 결과를 공유 가능한 이미지로 내보낼 수 있는 저장/공유 방식을 설계한다.
3. 모임 생성/초대 링크 플로우와 공동 페르소나 XP 프로토타입을 설계한다.
4. 하이브리드 페르소나 조건과 컬렉션 확장 규칙을 문서화한다.
5. Rive 또는 Lottie 런타임 도입 전 캐릭터 스타일 가이드와 에셋 입력값을 확정한다.
