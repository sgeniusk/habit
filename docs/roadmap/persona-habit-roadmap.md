# Persona Habit Roadmap

Updated: 2026-05-15

## 현재 상태 점검

Persona Habit은 현재 **모바일 웹 프로토타입** 단계다. 핵심 컨셉은 “생활 스냅을 남기면 페르소나가 성장하고, AI가 내가 모르는 습관까지 건강하게 읽어준다”로 정리되어 있다.

알파 개발 관리는 `docs/alpha/persona-habit-alpha-report.md`를 기준으로 한다. 현재 판단은 **알파 0.80 수준**이며, 다음 큰 빈칸은 공유 가능한 인증샷 결과물과 첫 사용자 온보딩이다.

완료된 기반:

- `오늘 / 스냅 / 집 / 모임 / 리포트` 5탭 프로토타입
- 생활 스냅, 위치/날씨 카드, 일기 작성 방식 선택
- 사진 미리보기, 필터/스티커/페르소나 도장 위치 오버레이, 새 기록 저장으로 이어지는 스냅 플로우
- 스냅 기록, 페르소나 애칭, 방/의상 꾸미기, 인증 도장/위치 선택의 로컬 저장
- 한국어 우선 다국어 기반, 영어 핵심 라벨, 언어 선택 저장
- 다양한 페르소나, 집 화면, 방/페르소나 꾸미기 방향
- 최신 스냅 카테고리에 따라 집 대표 페르소나가 바뀌는 반응 루프
- 규칙 기반 페르소나 성장 및 숨은 습관 인사이트
- 기록 기반 AI 모임 제안과 오래된 기억 큐레이션
- `SnapRecord`, 샘플 기록, 페르소나 카탈로그 데이터 경계 분리
- `src/views/`, `src/components/` 기반 화면/공용 UI 컴포넌트 분리
- 한글/영문 PRD
- 10명 가상 테스터 평가 보고서 검토
- 제작 에이전트 조직도
- GitHub 공개 저장소, Vercel 배포, GitHub Actions CI
- 로컬 하네스: lint, format, test, build, audit

아직 비어 있는 핵심:

- 실제 사진 편집 결과물의 영구 저장과 공유용 이미지 내보내기
- 첫 사용자 30초 온보딩과 첫 스냅 미션
- AI 인사이트가 왜 나왔는지 보여주는 한 줄 근거
- 페르소나 카탈로그와 성장 엔진의 XP/해금 아이템 통합
- 실제 위치/날씨 제공자 연결과 거리 계산
- 일기 타임라인과 캘린더 진입
- 실제 초대 링크 라우팅, 네이티브 공유, 공동 페르소나 상태 영속화
- 모임/리포트/페르소나 말투까지 확장된 전체 다국어 사전
- 네이티브 iOS/Android 전환 계획

## 제품 원칙

1. **기록보다 스냅**: 사용자가 검사받는 느낌이 아니라 오늘의 순간을 가볍게 남기는 느낌이어야 한다.
2. **성장보다 정체성**: 레벨 숫자보다 “나는 어떤 생활 페르소나가 되어가고 있는가”가 먼저 보여야 한다.
3. **AI보다 건강한 해석**: AI는 단정하거나 진단하지 않고, 반복되는 환경과 선택을 부드럽게 비춰줘야 한다.
4. **경쟁보다 같이 하기**: 모임은 압박형 챌린지가 아니라 다시 돌아오기 쉬운 사회적 루프여야 한다.
5. **웹 프로토타입에서 네이티브로**: 지금은 흐름 검증, 이후 Expo 기반 iOS/Android로 확장한다.
6. **가설과 검증 구분**: 가상 테스터 평가는 우선순위 가설로 쓰되, 실제 사용자 인터뷰와 행동 데이터로 다시 확인한다.

## 단계별 로드맵

### Phase 0. Prototype Stabilization

목표: 현재 프로토타입을 다음 기능을 얹을 수 있는 구조로 정리한다.

상태: **데이터/화면 경계 안정화 완료**. `App.tsx`는 탭 상태와 기록 저장 흐름을 관리하고, 화면 구현은 `src/views/`, 반복 UI는 `src/components/`로 분리했다.

- `src/App.tsx`의 타입, 카탈로그, 샘플 상태를 분리
- `SnapRecord`를 공유 타입으로 정리하고 `VerificationRecord`는 임시 호환 alias로 유지
- 페르소나 카탈로그를 별도 데이터 모듈로 분리
- `오늘 / 스냅 / 집 / 모임 / 리포트` 화면을 뷰 컴포넌트로 분리
- 페르소나 아바타, 메트릭, 기록 행, 모임 행을 공용 컴포넌트로 분리
- 테스트가 탭, 스냅, 집, 리포트 핵심 흐름을 계속 보증하게 유지
- 문서와 하네스 상태를 최신화

완료 기준:

- 기존 UI는 유지된다.
- 테스트와 빌드가 통과한다.
- 다음 기능이 `App.tsx` 한 파일에 계속 누적되지 않는다.

### Phase 1. Snap Loop MVP

목표: 사용자가 스냅을 찍고 꾸미고 올렸다는 느낌을 만든다.

상태: **MVP 구현**. 사진 미리보기, 필터/스티커 선택, 시간/횟수/페르소나 인증 도장, 도장 위치 선택, 프리뷰 오버레이, 새 기록 저장, 최근 기록 썸네일까지 동작한다. 새로고침 뒤에도 스냅 기록과 도장 위치가 유지된다.

- 이미지 선택 후 파일명뿐 아니라 미리보기 표시 완료
- 선택한 필터/스티커를 스냅 프리뷰 위에 시각적으로 표시 완료
- 시간 도장, 오늘 횟수 도장, 페르소나 도장 토글 구현 완료
- 페르소나 도장 위치 네 귀퉁이 선택과 저장 완료
- 인증샷 앱 벤치마킹 문서화 완료
- 필터 선택 상태와 스티커 선택 상태 구현 완료
- 선택한 이미지/필터/스티커/카테고리/장소/감정이 새 스냅 기록으로 저장 완료
- 스냅 기록 localStorage 저장 완료
- 최근 기록에 새 스냅이 즉시 반영 완료
- 사진 권한과 파일 선택 실패 상태 문구 정리

완료 기준:

- 사용자가 스냅을 남기면 오늘 기록, 리포트, 페르소나 성장에 반영된다.
- “꾸며서 올리기”가 단순 버튼이 아니라 실제 기록 루프로 느껴진다.

### Phase 2. Persona Home Loop

목표: 집 탭을 앱의 감정적 보상 중심으로 만든다.

상태: **부분 구현**. 최신 스냅 카테고리 기반 대표 페르소나 반응, 기록 기반 XP/레벨/해금 보상, 방/의상 꾸미기 선택 상태 저장, 페르소나 애칭과 직업 분리, 애칭 기반 대화는 연결되었다. 하이브리드 페르소나와 에셋 교체형 애니메이션 연결이 남아 있다.

- 최신 스냅 카테고리에 따라 대표 페르소나 행동 변경 완료
- XP, 레벨, 해금 아이템을 실제 기록 기반으로 표시 완료
- 방 꾸미기와 페르소나 꾸미기 아이템 선택 상태 구현 완료
- 페르소나 애칭 입력과 `학습자/러너/식단러` 직업 배지 구현 완료
- 페르소나 애칭, 방/의상 꾸미기, 인증 도장 선택 localStorage 저장 완료
- 학습 XP 기반 `척척박사 페르소나` 진화명 구현 완료
- Rive/Lottie/Spine 기반 애니메이션 에셋 파이프라인 문서화 완료
- 하이브리드 페르소나 조건 정의
- 페르소나 컬렉션을 카테고리별로 확장

완료 기준:

- 공부, 운동, 식단, 독서, 정리, 셀프케어 스냅이 각기 다른 페르소나 반응을 만든다.
- 집 탭에 들어갈 이유가 “구경”이 아니라 “내가 키운 결과 확인”이 된다.

### Phase 3. Today Journal And Context

목표: 오늘 탭을 하루 기록의 시작점으로 만든다.

상태: **부분 구현**. 한 줄 일기 입력, AI랑 같이쓰기/혼자 기록하기 모드, 날씨/습도/거리 기반 페르소나 대화, 규칙 기반 정리 초안이 들어갔다. 위치/날씨 카드는 허용, 거부, 실패 상태를 전환할 수 있고 `weatherEngine`에 날씨 어댑터 경계를 만들었다.

- 위치/날씨 카드의 허용, 거부, 실패 상태 프로토타입 완료
- 날씨 데이터 어댑터 인터페이스 초안 완료
- AI랑 같이쓰기 / 혼자 기록하기 선택 이후 화면 추가 완료
- 스냅, 날씨, 장소, 감정 기반 일기 초안 생성 흐름 초기 구현
- Day One류 일기 앱을 참고한 타임라인/캘린더 진입 구조 설계

완료 기준:

- 오늘 탭이 단순 대시보드가 아니라 “오늘을 남기는 입구”가 된다.

### Phase 4. Meet Social Loop

목표: 스투디처럼 친구를 초대하고 같이 지속하는 유인을 만든다.

상태: **MVP 구현**. `socialEngine`이 사용자의 반복 스냅을 읽고 러닝, 도서관 공부, 식단 모임을 제안하는 규칙 기반 추천 카드를 모임 탭에 연결했다. 제안 CTA에서 초대 링크, 복사/공유 문구, 수락자 저장, 대기실 첫 스냅 미션, 공동 페르소나 XP 성장을 확인할 수 있다. `/invite/:token` 라우트로 초대 수락 화면에 바로 진입할 수 있고, 모임 멤버와 공동 페르소나 XP, 미션 상태는 로컬 저장으로 유지된다. 네이티브 공유 시트와 서버 저장은 남아 있다.

- 기록 기반 모임 제안 카드 추가 완료
- 초대 링크 생성과 수락 대기 상태 미리보기 완료
- 초대 링크 복사와 공유 문구 상태 완료
- 초대 수락자 저장 상태와 첫 스냅 미션 완료
- 실제 초대 링크 라우트와 수락 화면 연결 완료
- 모임 멤버, 공동 XP, 미션 완료 상태 로컬 저장 완료
- 모임별 테마: 공부, 식단, 러닝, 혼합 생활 일부 구현
- 모임 생성/초대 링크 플로우 프로토타입
- 각자의 스냅이 공동 페르소나 XP에 기여
- 경쟁보다 복귀를 유도하는 가벼운 참여 상태
- 모임 리포트와 공동 페르소나 분위기 변화

완료 기준:

- 친구를 초대해야 할 이유가 보인다.
- 모임이 채팅방이 아니라 생활 스냅을 지속하게 하는 장치로 느껴진다.

### Phase 5. AI Habit Intelligence

목표: “내가 모르는 습관까지 AI가 발견한다”는 핵심 약속을 구현 가능한 형태로 좁힌다.

상태: **초기 구현**. 리포트는 규칙 기반 인사이트를 보여주되, 사용자가 문구를 부드럽게 바꾸거나 관심 없는 분석을 숨기고 다시 복구할 수 있다. 이 피드백은 로컬 저장되어 다음 방문에도 유지된다. 오래된 기억 모드는 월별, 장소별, 페르소나별 필터와 회상 질문으로 “오래전의 나”를 다시 꺼낸다. 실제 모델 연결 전까지 단정적인 표현을 낮추고 사용자 피드백을 먼저 쌓는 방향으로 간다.

- 인사이트 타입: 장소 패턴, 시간대 패턴, 카테고리 균형, 회복 루틴, 하이브리드 정체성
- 안전 문구와 금지 문구 정의
- 사용자가 불편해할 수 있는 분석의 숨김/수정/피드백 플로우 초기 구현
- AI 인사이트 피드백 로컬 저장 완료
- AI 인사이트별 한 줄 근거 표시 추가 예정
- AI 일기 초안과 습관 인사이트의 데이터 경계 분리
- 오래된 스냅을 월별 기억 카드와 회상 질문으로 큐레이션하는 리포트 모드 초기 구현
- 오래된 기억을 월별, 장소별, 페르소나별로 필터링하는 기록 탐색 UI 초기 구현
- 실제 모델 연동 전 규칙 기반 시뮬레이션 강화

완료 기준:

- AI가 무섭거나 단정적으로 느껴지지 않고, 사용자가 스스로를 이해하는 데 도움을 준다.

### Phase 6. Native App Readiness

목표: 웹 프로토타입의 검증된 흐름을 iOS/Android 출시 계획으로 옮긴다.

상태: **전환 계획 문서화**. `docs/native/expo-readiness-plan.md`에 Expo 기반 앱 구조, 권한 정책, 저장소/백엔드 후보, 앱스토어 제출 전 최소 기능, 웹과 네이티브가 공유할 도메인 로직 목록을 정리했다. 웹 프로토타입은 브라우저 위치 API와 Open-Meteo 형식의 현재 날씨 응답을 읽는 어댑터를 갖췄고, 사용할 수 없을 때는 데모 날씨로 안전하게 돌아간다.

- Expo React Native 구조 초안 문서화
- 카메라, 사진첩, 위치, 푸시 알림 권한 정책 문서화
- 브라우저 위치 API와 날씨 제공자 후보 어댑터 초기 구현
- 로컬 저장, 계정, 백엔드 선택지 정리
- 앱스토어 제출 전 최소 기능 정의
- 웹과 네이티브의 공유 가능한 도메인 로직 분리 목록 작성

완료 기준:

- “웹에서 예쁜 프로토타입”을 넘어 실제 앱 출시 계획이 선다.

### Phase 7. Internationalization Foundation

목표: 한국어 UX를 유지하면서 영어와 이후 일본어 확장에 대비한다.

상태: **초기 구현**. `src/lib/i18n.ts`에 `ko/en` 번역 사전과 locale formatter 경계를 만들었다. 언어 선택은 사용자 설정에 저장되고, 앱을 다시 열어도 유지된다. 탭, 오늘/스냅 핵심 문구, 카테고리/장소 라벨은 영어로 전환된다.

- `Locale = "ko" | "en"` 타입 추가 완료
- 언어 선택 UI와 저장 완료
- 하단 탭, 오늘 제목, 스냅 제목/CTA/카테고리/장소/인증 도장 다국어화 완료
- `document.documentElement.lang` 동기화 완료
- 다국어 계획 문서화 완료
- 모임/리포트/페르소나 말투 사전 확장
- 날짜, 시간, 거리, 온도 formatter 분리

완료 기준:

- 앱의 핵심 루프가 한국어/영어 모두에서 의미가 통한다.
- 신규 문구는 화면 컴포넌트가 아니라 번역 사전에 먼저 추가된다.

### Phase 8. Alpha Evaluation Response

목표: 10명 가상 테스터 평가 보고서를 알파 우선순위에 반영한다.

상태: **검토 완료**. 보고서는 실제 인터뷰가 아니라 시뮬레이션이므로 정량 점수는 가설로만 사용한다. 하지만 온보딩, 공유용 인증샷, AI 근거 표시, 공동 페르소나 중심 모임, 말투/테마 선택은 기존 알파 방향과 일치한다.

- 평가 보고서 문서 목록 포함 완료
- P0: 첫 사용자 30초 온보딩
- P0: 공유용 인증샷 이미지
- 완료: 페르소나 도장 위치 선택
- P0: AI 인사이트 한 줄 근거 표시
- P1: 시험 기간 모드, 과목/D-Day 태그
- P1: 귀여운 톤과 차분한 톤 선택
- P1: 공동 페르소나 꾸미기와 복귀 UX

완료 기준:

- 첫날 사용자가 첫 스냅, 페르소나 반응, 공유 또는 리포트 확인까지 한 번에 경험한다.
- 가상 평가에서 나온 P0 항목이 실제 앱 플로우에 들어간다.

## 우선순위 판단

다음 개발 순서는 다음과 같다.

1. **구조 안정화**: 다음 기능이 들어갈 자리 만들기
2. **스냅 루프**: 사진을 남기고 꾸미고 기록되는 감각 만들기
3. **집 보상 루프**: 기록이 페르소나 행동과 꾸미기로 이어지게 만들기
4. **오늘 일기 루프**: 하루 회고와 AI 보조 작성으로 확장
5. **모임 루프**: 친구 초대와 공동 페르소나로 소셜 동기 강화
6. **AI/네이티브 준비**: 실제 서비스로 넘어가기 위한 기반 설계
