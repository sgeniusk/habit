# Persona Habit PRD 한국어판

## 제품 정의

Persona Habit은 생활 인증샷을 기반으로 사용자가 스스로 인식하지 못하는 습관과 생활 패턴을 발견하고, 이를 더 건강한 루틴으로 바꿔가도록 돕는 앱이다.

사용자는 공부, 식단, 운동, 독서, 정리, 셀프케어 같은 일상 행동을 사진으로 인증한다. 앱은 이 기록을 바탕으로 여러 페르소나를 성장시키고, 친구와 함께 키우는 생활방을 만들며, AI가 숨은 습관과 개선 방향을 제안한다.

작업명: **Persona Habit**

한국어 포지셔닝: **내가 모르는 습관까지 발견해서 건강하게 키워주는 생활 인증 앱**

## 문제 정의

기존 습관앱은 대부분 체크리스트 중심이다. 사용자는 할 일을 체크하지만, 체크 자체가 충분히 재미있거나 감정적인 보상이 되지는 않는다. 생활기록앱은 더 많은 정보를 남길 수 있지만, 사용자가 꾸준히 기록해야 할 이유가 약하다.

또한 사람들은 자신의 생활 패턴을 의외로 잘 모른다. 어느 장소에서 공부가 잘 되는지, 어느 시간대에 군것질을 하는지, 운동을 혼자 할 때보다 친구와 할 때 더 잘 지속하는지, 어떤 환경에서 좋은 습관이 반복되는지 같은 정보는 기록이 쌓인 뒤에야 보인다.

Persona Habit은 이 문제를 **사진 인증**, **캐릭터 성장**, **AI 패턴 분석**, **친구와의 공동 루틴**으로 해결한다.

## 타깃 사용자

- 공부 루틴과 인증 동기가 필요한 학생, 수험생
- 식단, 운동, 공부, 자기관리를 균형 있게 만들고 싶은 직장인
- 친구와 가볍게 인증하며 서로 동기부여를 받고 싶은 그룹
- 귀여운 캐릭터 수집, 꾸미기, 성장 요소를 좋아하는 사용자

## 핵심 가치

사용자는 인증샷을 찍을 때마다 세 가지 보상을 받는다.

1. 즉각적 보상: 페르소나 XP, 레벨, 의상, 방 꾸미기 아이템, 수집 요소
2. 사회적 보상: 친구와 함께 생활방을 만들고 공동 페르소나를 성장시키는 경험
3. 성찰적 보상: AI가 숨은 생활 패턴을 발견하고 건강한 다음 행동을 제안

이 앱의 핵심은 “기록하라”가 아니라 “인증하면 나의 페르소나가 자란다”는 동기 구조다.

## MVP 범위

### 1. 생활 인증

사용자는 하나의 인증 기록을 만들 때 다음 정보를 남긴다.

- 사진 업로드 또는 카메라 촬영
- 카테고리: 공부, 식단, 운동, 독서, 정리, 셀프케어, 취미
- 장소 유형: 집, 도서관, 학교, 카페, 헬스장, 식당, 야외, 기타
- 선택 입력: 한 줄 메모
- 자동 입력: 인증 시간

첫 프로토타입에서는 브라우저 파일 입력을 통해 카메라 촬영 또는 사진 선택을 시뮬레이션한다. 네이티브 카메라 API는 실제 앱 단계에서 구현한다.

### 2. 페르소나 컬렉션

사용자는 하나의 캐릭터만 키우는 것이 아니라 여러 페르소나를 소유한다. 반복되는 인증 기록에 따라 특정 페르소나가 생성되거나 성장한다.

- 공부 기록이 쌓이면 공부 페르소나가 성장
- 러닝, 운동 기록이 쌓이면 활동형 페르소나가 성장
- 식단 기록이 쌓이면 영양/식습관 페르소나가 성장
- 여러 습관이 섞이면 “공부하는 러너”, “건강관리형 수험생” 같은 하이브리드 페르소나로 확장

각 페르소나는 다음 속성을 가진다.

- 레벨
- XP
- 성향 태그
- 성장 이력
- 해금된 의상과 방 꾸미기 아이템
- 진화 단계
- 가벼운 움직임 상태: 숨쉬기, 눈 깜빡임, 손 흔들기, 아이템 튐, 방 분위기 반응

페르소나는 동영상 파일이 아니라 UI 레이어 기반 캐릭터로 만든다. 그래야 습관 카테고리, 장소, 친구방 상태, 진화 단계에 따라 실시간으로 반응할 수 있다.

### 3. 숨은 습관 AI

AI는 반복되는 인증 기록을 분석해 사용자가 직접 의식하지 못한 패턴을 보여준다.

예시:

- 강한 장소 패턴: “도서관에서 공부 인증이 가장 안정적으로 반복돼요.”
- 약한 시간대: “늦은 밤 식단 인증이 긴 공부일 뒤에 몰려 있어요.”
- 건강한 제안: “공부 인증이 2회 이상 있는 날에는 밤 루틴을 가볍게 설계해 보세요.”
- 정체성 피드백: “당신은 환경을 잘 설계할수록 꾸준해지는 학습자에 가까워요.”

MVP 프로토타입에서는 미리 정의된 규칙 기반 인사이트를 보여준다. 실제 서비스에서는 사용자 데이터, 개인정보 보호, 동의 플로우가 준비된 뒤 AI 분석 파이프라인을 붙인다.

### 4. 생활방과 소셜 루프

사용자는 혼자 페르소나를 키울 수도 있고, 친구와 함께 생활방을 만들 수도 있다.

- 공부방, 식단방, 러닝방, 혼합 생활방 생성
- 초대 링크로 친구 참여
- 각자의 인증샷이 방 XP에 기여
- 방의 행동 패턴에 따라 공동 페르소나의 성격과 분위기가 변화
- 부담을 주는 경쟁보다, 가볍게 참여하고 다시 돌아올 수 있는 연속성 중심 설계

생활방의 역할은 단순 채팅이 아니라 **인증을 지속하게 만드는 사회적 장치**다.

### 5. 리포트

앱은 주기적으로 다음 리포트를 제공한다.

- 7일 생활 리포트
- 페르소나 성장 요약
- 카테고리 균형
- 장소와 시간대 패턴
- AI 기반 습관 추천

리포트는 “얼마나 많이 했는가”보다 “어떤 상황에서 잘 지속되는가”를 보여주는 데 초점을 둔다.

## MVP에서 제외할 것

첫 MVP에서는 다음 기능을 제외한다.

- 실제 결제와 구독
- 완전한 AI 이미지 인식
- 가짜 인증 감지
- 네이티브 iOS/Android 배포
- 백그라운드 GPS 추적
- 외부 생활기록 앱과의 데이터 연동

## 네이티브 앱 방향

실제 iOS/Android 앱은 Expo React Native 기반으로 만드는 것이 적합하다. 이유는 다음과 같다.

- 카메라 촬영 지원
- 사진첩 접근 지원
- 푸시 알림 지원
- 위치 권한 확장 가능
- App Store, Play Store 배포 가능
- React 기반 코드를 일부 공유 가능

현재 프로토타입은 네이티브 개발 전에 제품 흐름과 시각 방향을 검증하기 위한 모바일 웹앱이다.

## 프로토타입 요구사항

프로토타입은 모바일 앱처럼 보이는 5개 탭 구조를 가진다.

- 오늘: 오늘의 인증 유도, 연속 기록, 최근 인증, 대표 숨은 습관
- 인증: 카테고리/장소 선택, 사진 업로드 또는 촬영 상태
- 페르소나: 보유 페르소나, 레벨, XP, 진화, 꾸미기 아이템
- 방: 친구방 상태, 공동 페르소나, 그룹 인증 루프
- 리포트: 7일 AI 패턴 요약과 건강한 다음 행동

프로토타입은 랜딩페이지가 아니라 실제 앱 화면처럼 느껴져야 한다. 사용자가 10초 안에 “사진 인증을 하면 캐릭터가 자란다”는 핵심 구조를 이해해야 한다.

## 성공 기준

- 사용자가 10초 안에 인증샷과 페르소나 성장의 관계를 이해한다.
- 사용자가 공부, 식단, 운동 같은 생활 인증을 자연스럽게 상상할 수 있다.
- 여러 페르소나 보유, 성장, 꾸미기, 진화가 눈에 보인다.
- 생활방이 단순 채팅이 아니라 인증 지속을 돕는 소셜 루프로 느껴진다.
- AI 인사이트가 생활 기록과 건강한 습관 개선을 명확히 연결한다.
